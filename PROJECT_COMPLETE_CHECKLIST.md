# PixelPlay 기본 채팅 프로젝트 완료 체크리스트

이 문서는 심화 스트리밍 기능을 제외하고, 현재 PixelPlay의 기본 실시간 채팅 웹 애플리케이션 완성도를 점검하기 위한 팀 공유용 체크리스트입니다.

## 상태 라벨

- ✅ 완료: 요구사항 기준으로 동작 가능한 상태
- 🟡 보완 필요: 기본 구현은 있으나 권한, 상태 분기, 캐싱, UX 등 보완 필요
- ❌ 미구현: 기본 요구사항 기준 기능 없음
- 🚨 치명: 보안, 권한, 서비스 성립에 즉시 영향

## 중요도 기준

- P0: RLS, 권한 제어처럼 서비스 보안/핵심 동작에 치명적인 항목
- P1: 기본 프로젝트 필수 요구사항
- P2: 기본 기능 안정화와 정합성 보완
- P3: 기본 요구사항 밖의 선택/추후 협의 기능

## unread와 나가기 책임 분리

- unread는 `chat_room_member.last_read_at` 기반으로 처리합니다.
- `last_read_at`은 채팅방 상세 페이지 진입/이탈, 하단 도달, 또는 읽음 기준 이벤트에서 갱신하는 것이 적절합니다.
- 채팅방 나가기는 unread와 별도 책임입니다.
- 나가기는 멤버십 상태, 재입장 흐름, `last_joined_at`, `is_banned` 정책과 연결해서 설계합니다.
